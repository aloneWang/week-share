## ASCII(American Standard Code for Information Interchange)
### 释义：
一种涵盖了 数字，字母，控制字符的单字节编码规则，
用7位来(第八位为0)表示所有的数字，大小字母，控制字符，总共128个，也就是单字节（一个字节8位）
后128个是`扩展ASCII码`用来表示外来语和图形符号

### 产生
对于计算机所有的存储和运算都用二进制表示，所以对于人类的字符就要通过转义转为二进制计算机才能识别

### 问题
由于字符的增多，`ASCII码`不够用了, 就出现了`GBK, Unicode, GB2312`等

## Unicode
是一种字符集，收纳了全世界所有的字符，开始是 `UCS-2版本` 即每个字符用两个字节表示也就是16位，所以收纳空间是[0-65535],
`charCodeAt`返回的codePoint(代码点) 的取值范围就是`[0-65535]`(非负整数)

### 发展
随着互联网的发展，6万多个字符已经不够表示各个国家的字符，所以有了 `USC-4`,即用4个字节来表示，所以空间位[0-4294967296],这个大小已经够够了，最新的 [Unicode 13.0](https://www.unicode.org/versions/Unicode13.0.0/), 已经收录了143,859 个字符，这时候用chartCodeAt返回就不行了，得用 `codePointAt`

## utf-8
utf 全称 `Unicode Transformation Format` , Unicode 只是将字符 和 非负整数(`codePoint`)关联起来，它的unicode值只是一个数字，并没有将字符存储起来，最为常见的存储方式（编码方式） 是 `utf-8， utf-16`,

### utf-8 编码方式
utf-8 采用 1-4 个 字节表示 一个 Unitcode 值(而一个unitCode值对应一个字符)，而且是兼容 `ASCII`编码，
有些情况下相对于 `utf-16` 更节省空间

### utf-16 编码方式
utf-16 , 16 位也就是2个字节，所以 采用 2 或 4个字节表示一个 Unicode，对于 ASCII 文件采用 utf-16 编码，会有很多的00字节，造成空间浪费，毕竟 ASCII 码 是可以用1个字节表示的，所以使用 `utf-8`更为广泛些

### 针对 中文unicode 与 utf-8 字节大小
中文的unitcode 大小是 16位 也就是2字节， 而utf-8编码 的中文是3字节
utf-8 规则：
+ 1个字节的字符第一位定死0， 所以就有128个字符，兼容ASCII
+ n(n>1)个字节，则第 2 个字节到 第n个字节前2位都定死10， 第一个字节前n+1位 是n个1 和 一个0

所以utf-8的3字节 减去规则站位 24-8=16，刚好是2字节
